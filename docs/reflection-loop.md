# Self-Reflection Loop

The reflection loop is the part that makes Allen Memory OS more than a passive archive.

## Daily Automation

Each night:

1. Gather the day's transcript events.
2. Generate a daily summary.
3. Extract long-term memory candidates.
4. Compare new candidates with existing memories.
5. Write a reflection note.
6. Produce questions for uncertain memories.

## Reflection Output

The system should keep four kinds of reflection:

- Observation: what happened today.
- Pattern: what seems repeated across days.
- Hypothesis: what might be true about the user but needs more evidence.
- Improvement: what the system should change about its own behavior.

## Memory Promotion Rules

Promote a record when it is:

- tied to a goal, project, decision, habit, or relationship
- repeated across multiple days
- explicitly marked important by the user
- useful for future planning or decision-making

Do not promote when it is:

- private noise with no future value
- a one-off low-confidence inference
- contradicted by newer evidence
- sensitive content that the user configured for short retention

## Future Agent Behavior

When unsure, the agent should ask:

- Should I remember this long term?
- Is this still true?
- Did your opinion change?
- Is this project active, paused, or abandoned?
